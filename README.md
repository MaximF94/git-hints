# Подсказки себе для работы с GIT
----
создать папку: mkdir "имя папки"
перейти в эту папку: cd "имя папки"
создать файл: touch "имя файла"
добавить контроль версий для данной папки: git init
посмотреть статус файлов в git: git status
добавить все файлы в контроль версий (предподгтовка): git add --all
добавить выборочно файлы в контроль версий: git add "file1" "file2"
сделать commit (создать версию проекта): git commit -m "Комментарий"
в github создаем проект и копируем ссылку на него
объеденить локальный проект и проект на github: git remote add origin ссылка на проект
добавить изменения из локальной дирректори на github: git push -u origin master (если проект добавляется в первый раз, то ставим -u, при последующих push не ставим)

статус untracked - не осталеживаемый. То есть это новый файл, который не был добавлен в систему контроля версий. Чтобы он был добавлен нужно выполнить команду git add
статус staged - файл был добавлен git add и попадает в попадает в staging area. То есть файл войдет в commit
статус tracked - после выполнения git commit файл становится отслеживаемым
статус modified - после того, как файл, который был добавлен и выполнен commit, изменили, git начинает сравнивать с последней версией. если файлы отличаются, то данному файлу присваивается этот статус.
